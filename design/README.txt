CARPETA DESIGN - DESCRIPCIÓN
=============================

Esta carpeta contiene los archivos de diseño y modelado del sistema de procesamiento de datos.

CONTENIDO:
----------

ARCHIVOS DE DISEÑO:
- 1A.mdj: Archivo de modelo UML creado con StarUML o herramienta similar
  * Contiene diagramas de clases del sistema
  * Muestra las relaciones entre las diferentes clases
  * Documenta la arquitectura del software

PROPÓSITO:
----------
Esta carpeta almacena toda la documentación visual y de diseño del proyecto:
1. Diagramas UML (clases, secuencia, casos de uso)
2. Modelos de arquitectura del sistema
3. Especificaciones de diseño
4. Documentación gráfica de la estructura del software

TIPOS DE DIAGRAMAS INCLUIDOS:
-----------------------------
- Diagrama de Clases: Muestra las clases del sistema y sus relaciones
- Estructura del paquete Model con todas sus clases
- Relaciones de dependencia entre componentes
- Métodos y atributos de cada clase

HERRAMIENTAS UTILIZADAS:
------------------------
- StarUML: Para creación de diagramas UML
- Formato .mdj: Formato nativo de StarUML

IMPORTANCIA EN EL DESARROLLO:
-----------------------------
- Facilita la comprensión de la arquitectura del sistema
- Ayuda en la planificación del desarrollo
- Documenta las decisiones de diseño
- Sirve como referencia para futuras modificaciones
- Permite comunicar el diseño a otros desarrolladores

ACTUALIZACIÓN:
--------------
Los diagramas deben actualizarse cada vez que se modifique la estructura
de clases o se agreguen nuevas funcionalidades al sistema.

ACCESO:
-------
Para visualizar los archivos .mdj se requiere StarUML u otra herramienta
compatible con este formato de modelado UML.
