CARPETA PM (PROJECT MANAGEMENT) - DESCRIPCIÓN
=============================================

Esta carpeta contiene todos los documentos y archivos relacionados con la gestión y administración del proyecto.

PROPÓSITO:
----------
Centraliza la documentación de gestión del proyecto:
1. Seguimiento y control del progreso
2. Reportes de estado y avance
3. Gestión de riesgos y issues
4. Comunicaciones del proyecto
5. Documentación administrativa

TIPOS DE ARCHIVOS ESPERADOS:
-----------------------------
- Reportes de progreso y status
- Actas de reuniones y revisiones
- Logs de seguimiento de tareas
- Documentos de control de cambios
- Reportes de métricas del proyecto
- Comunicaciones oficiales
- Documentos de cierre de fases

ÁREAS DE GESTIÓN CUBIERTAS:
---------------------------
1. GESTIÓN DEL ALCANCE:
   - Control de cambios en requisitos
   - Documentación de scope creep
   - Aprobaciones de modificaciones

2. GESTIÓN DEL TIEMPO:
   - Seguimiento de cronograma
   - Reportes de desviaciones
   - Análisis de cumplimiento de fechas

3. GESTIÓN DE CALIDAD:
   - Reportes de métricas de calidad
   - Documentos de revisiones
   - Planes de mejora

4. GESTIÓN DE RIESGOS:
   - Registro de riesgos identificados
   - Planes de mitigación
   - Seguimiento de riesgos materializados

RELACIÓN CON PSP:
-----------------
En el Personal Software Process, esta carpeta incluye:
- Logs de tiempo personal (Time Recording Log)
- Logs de defectos (Defect Recording Log)
- Reportes de métricas personales
- Análisis de productividad
- Documentación de mejoras al proceso personal

ESTRUCTURA RECOMENDADA:
-----------------------
pm/
├── reports/        # Reportes de progreso
├── meetings/       # Actas y minutas
├── metrics/        # Métricas del proyecto
├── risks/          # Gestión de riesgos
├── communications/ # Comunicaciones oficiales
└── closure/        # Documentos de cierre

HERRAMIENTAS DE GESTIÓN:
------------------------
- Hojas de cálculo para seguimiento
- Herramientas de gestión de proyectos
- Sistemas de tracking de issues
- Dashboards de métricas

IMPORTANCIA:
------------
- Facilita el control y seguimiento del proyecto
- Proporciona visibilidad del progreso
- Permite la toma de decisiones informadas
- Documenta lecciones aprendidas
- Mejora la gestión de futuros proyectos

ESTADO ACTUAL:
--------------
La carpeta está preparada para recibir documentos de gestión.
Se recomienda establecer una estructura organizativa clara desde el inicio.

MANTENIMIENTO:
--------------
- Actualizar reportes regularmente
- Mantener métricas actualizadas
- Documentar decisiones importantes
- Archivar documentos por períodos o fases
