CARPETA PLANNING - DESCRIPCIÓN
===============================

Esta carpeta contiene toda la documentación relacionada con la planificación del proyecto de software.

PROPÓSITO:
----------
Almacena los documentos de planificación y organización del proyecto:
1. Planes de desarrollo y cronogramas
2. Estimaciones de tiempo y esfuerzo
3. Definición de requisitos y especificaciones
4. Análisis de riesgos y mitigación
5. Distribución de tareas y responsabilidades

TIPOS DE ARCHIVOS ESPERADOS:
-----------------------------
- Planes de proyecto (.doc, .pdf, .txt)
- Cronogramas y calendarios
- Documentos de requisitos
- Estimaciones de esfuerzo y tiempo
- Análisis de factibilidad
- Matrices de trazabilidad
- Documentos de alcance del proyecto

CONTENIDO TÍPICO:
-----------------
- Project Charter (Acta de constitución del proyecto)
- Work Breakdown Structure (WBS)
- Estimaciones PSP (Personal Software Process)
- Cronograma de actividades
- Definición de entregables
- Criterios de aceptación
- Plan de calidad

RELACIÓN CON PSP:
-----------------
En el contexto del Personal Software Process:
- Contiene las estimaciones iniciales del proyecto
- Documenta la planificación personal del desarrollo
- Incluye métricas de planificación
- Registra los compromisos de tiempo y calidad

FASES DE PLANIFICACIÓN:
-----------------------
1. Análisis de requisitos
2. Estimación de tamaño
3. Estimación de esfuerzo
4. Planificación temporal
5. Definición de calidad
6. Análisis de riesgos

IMPORTANCIA:
------------
- Establece las bases del proyecto
- Facilita el seguimiento y control
- Permite la evaluación de desviaciones
- Mejora la precisión de futuras estimaciones
- Documenta las decisiones de planificación

ESTADO ACTUAL:
--------------
La carpeta está preparada para recibir documentos de planificación.
Se recomienda organizar los archivos por fases o tipos de documento.

MANTENIMIENTO:
--------------
- Actualizar planes según cambios en el proyecto
- Mantener versiones históricas para análisis
- Documentar desviaciones y lecciones aprendidas
- Revisar y refinar estimaciones regularmente
